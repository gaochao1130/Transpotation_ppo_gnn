# 空箱运输智能调度优化

## 项目简介
本项目聚焦物流场景下的空箱运输调度问题，旨在通过优化算法实现空箱从源站到目的站的高效调配。在满足所有需求的前提下，最小化总运输里程，从而降低物流成本，提升运作效率。

## 当前问题描述
- **场景设定**：存在多个源站（供应空箱）和目的站（需求空箱），且总供应量充足，能够满足所有目的站的需求。
- **优化目标**：最小化总运输里程（总里程为所有调运步骤的站间里程累加）。
- **核心权衡**：在制定调度策略时，主要面临以下博弈：
  1. **单次运输成本**：倾向于选择距离较近的站点进行调运（单次里程越小越好）。
  2. **总体调度步骤**：频繁的小批量调运会导致步骤数增加，从而推高累计里程。
  3. **供需匹配效率**：就近调运可能面临单次供应量不足的问题，导致需要额外次数的补充调运；而一次性满足大批量需求虽能减少调度次数，但可能因距离较远使得单次运输里程过大。
- **已知条件**：包括初始站间里程矩阵、各车站的地理分布（图结构）、以及各站点的实时需求与供应数据。

## 项目结构
我们提供了构建和测试算法所需的基础数据及脚本环境：
- `train.py`: 当前的主要调度算法逻辑。
- `基础数据/`: 包含各站点信息、供需数据（如 `demand_data_TOPK.json`, `supply_data_TOPK.json`）、距离矩阵（`distance_matrix.csv`）及图拓扑结构。

## 下一步优化方向：多箱型联合调度
计划将模型扩展至**2种及以上箱型的联合调度**场景：
- **箱型独立性**：不同箱型的需求与供应分布各自独立（例如，某站可能是 A 箱型的源站，同时也是 B 箱型的目的站）。
- **联合运输机会**：如果不同箱型存在相同起终点的运输需求，便可在同一调运步骤中合并发送，进而精简调度步骤。
- **利弊权衡**：合并运输能够有效减少整体步骤，但也可能改变系统状态，从而错失后续针对单个箱型的更优调度机会（即存在局部最优与全局最优的冲突）。
- **优化目标升级**：在允许且激励不同箱型联合发送的前提下，设计更具全局视野的调度策略，实现整体总运输里程的绝对最小化。
